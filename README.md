# Banking Client Retention Risk Dashboard

## Project Overview
This project focuses on assessing and improving client retention in banking by analyzing client data and applying various data science techniques to identify at-risk clients and develop dashboards for insights.

## Objectives
- To identify clients at risk of retention loss.
- To analyze key retention metrics and trends.
- To create visual dashboards that present findings effectively.

## Structure
- **data/**: Contains raw and processed data.
- **sql/**: SQL scripts for data extraction and manipulation.
- **dashboards/**: Final dashboards created using data visualization tools.
- **notebooks/**: Jupyter notebooks for exploratory data analysis.

## Data Pipeline
1. Data Collection: Collect data from banking systems and external sources.
2. Data Cleaning: Clean and preprocess data for analysis.
3. Data Transformation: Transform raw data into usable formats for analysis.
4. Analysis: Perform statistical analyses and model building to identify risk factors.
5. Visualization: Create dashboards to visualize results.

## SQL Scripts
The `sql/` directory contains various scripts for data extraction and transformation. Key scripts include:
- `client_data_extraction.sql`: Extracts raw client data from the database.
- `client_data_transformation.sql`: Transforms raw data to produce structured datasets for analysis.

## Dashboards
Dashboards created for this project visualize client retention metrics effectively. They include:
- Client Risk Dashboard: Presents metrics on retention risks.
- Executive Summary Dashboard: Provides an overview of project findings.

## Executive Summary
The aim of this project is to use data-driven approaches to improve client retention in banking. By identifying at-risk clients, the bank can take proactive measures to retain them and enhance customer satisfaction.

## Setup Instructions
1. Clone the repository: `git clone https://github.com/Liliam-Cordeiro/Banking-Client-Retention-Risk-Dashboard.git`
2. Install necessary dependencies: `pip install -r requirements.txt`
3. Set up the database and run SQL scripts to extract data.

## Usage Guidelines
- Navigate to the `notebooks/` directory to run data analysis notebooks.
- Access dashboards in the `dashboards/` directory for visual insights.
